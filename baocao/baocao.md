1.1.1	Vị trí và chức năng
Trung tâm Ngoại ngữ – Tin học Victory là đơn vị đào tạo trực thuộc Trường Đại học Trà Vinh, được thành lập từ năm 2006. Trung tâm có chức năng tổ chức các hoạt động đào tạo, bồi dưỡng ngoại ngữ, tin học và các kỹ năng ngắn hạn nhằm đáp ứng nhu cầu học tập của cán bộ, công chức, viên chức, sinh viên và học sinh trên địa bàn tỉnh Trà Vinh cũng như các khu vực lân cận.
Năm 2014, Trung tâm Victory chính thức trở thành Trung tâm Ủy quyền của Bộ 
phận Đánh giá Ngôn ngữ Cambridge (mã số VN370). Đây là bước ngoặt quan trọng, khẳng định vị thế của Trung tâm trong việc tổ chức các kỳ thi đánh giá năng lực ngôn ngữ theo tiêu chuẩn quốc tế tại địa phương. Với phương châm hoạt động “Tận tâm – Chất lượng – Hiệu quả”, Trung tâm luôn nỗ lực nâng cao chất lượng dịch vụ để đồng hành cùng người học trên con đường hội nhập.


1.1.2	Nhiệm vụ và quyền hạn
Trung tâm Victory thực hiện các nhiệm vụ trọng tâm sau:
-	Đào tạo Ngoại ngữ: Giảng dạy đa dạng các ngôn ngữ bao gồm tiếng Anh, tiếng Hoa, tiếng Nhật và tiếng dân tộc (tiếng Khmer). Chương trình đào tạo được xây dựng từ căn bản đến nâng cao, phù hợp với mọi lứa tuổi.
-	Đào tạo Tin học: Tổ chức các khóa bồi dưỡng, đào tạo và thi cấp chứng chỉ ứng dụng Công nghệ thông tin (cơ bản và nâng cao) theo tiêu chuẩn của Bộ Giáo dục và Đào tạo. Ngoài ra, Trung tâm còn triển khai các lớp Tin học dành riêng cho thiếu nhi.
-	Khảo thí quốc tế: Tổ chức các kỳ thi chứng chỉ quốc tế như Cambridge (Starters, Movers, Flyers, KET, PET...) và IELTS. Giảng dạy, tổ chức thi tiếng Anh theo Khung năng lực ngoại ngữ 6 bậc dùng cho Việt Nam (VSTEP).
-	Dịch vụ đào tạo theo nhu cầu: Triển khai các lớp học Premium (mô hình 1:1, 1:2, 1:3...) thiết kế riêng theo yêu cầu cụ thể của học viên về thời gian và lộ trình học tập.
-	Dịch vụ hỗ trợ khác: Thực hiện các dịch vụ dịch thuật, hợp pháp hóa lãnh sự, tư vấn du học và cung cấp các sản phẩm quà lưu niệm liên quan đến môi trường giáo dục.
-	Nghiên cứu và phát triển: Thường xuyên cập nhật, đổi mới phương pháp giảng dạy và giáo trình theo xu hướng quốc tế nhằm đảm bảo chất lượng đào tạo tốt nhất cho học viên.
1.1.3	Thế mạnh của đơn vị
-	Đội ngũ nhân sự: Sở hữu đội ngũ giáo viên giỏi, trẻ trung, năng động và tận tâm. Đặc biệt, Trung tâm có sự tham gia giảng dạy của giáo viên nước ngoài đúng chuyên môn, giúp học viên phát triển kỹ năng phát âm và giao tiếp chuẩn.
-	Cơ sở vật chất và uy tín: Là Trung tâm khảo thí ủy quyền duy nhất tại Trà Vinh từ năm 2014, sở hữu hạ tầng kỹ thuật và quy trình quản lý đạt chuẩn quốc tế của Cambridge.
-	Chương trình linh hoạt: Hệ thống khóa học đa dạng, học phí hợp lý, đáp ứng kịp thời nhu cầu thực tế của thị trường lao động và yêu cầu chuẩn đầu ra của sinh viên.

2.2.2	Công nghệ sử dụng
2.2.2.1	Giới thiệu Spring Boot Framework và React.js

**Spring Boot Framework**

Spring Boot là một framework Java mã nguồn mở được xây dựng trên nền tảng Spring Framework, giúp đơn giản hóa việc phát triển ứng dụng Java. Spring Boot tự động cấu hình các thành phần cần thiết, giảm thiểu code boilerplate và cho phép lập trình viên tập trung vào logic nghiệp vụ.

Đặc điểm nổi bật:
-	**Auto-configuration**: Tự động cấu hình các bean và dependency dựa trên classpath.
-	**Embedded Server**: Tích hợp sẵn Tomcat, Jetty hoặc Undertow, không cần deploy file WAR.
-	**Spring Data JPA**: Đơn giản hóa việc truy xuất cơ sở dữ liệu với ORM (Object-Relational Mapping).
-	**Spring Security**: Framework bảo mật mạnh mẽ hỗ trợ authentication và authorization.
-	**RESTful API**: Dễ dàng xây dựng API REST với annotation @RestController, @GetMapping, @PostMapping...
-	**Dependency Injection**: Quản lý dependency tự động, giảm coupling giữa các module.

**React.js**

React là thư viện JavaScript mã nguồn mở do Facebook phát triển, chuyên dụng cho việc xây dựng giao diện người dùng (UI). React sử dụng mô hình Component-Based, cho phép chia nhỏ giao diện thành các component độc lập, tái sử dụng được.

Đặc điểm nổi bật:
-	**Virtual DOM**: React sử dụng Virtual DOM để tối ưu hóa việc cập nhật giao diện, chỉ render lại những phần thay đổi.
-	**Component-Based**: Xây dựng UI từ các component nhỏ, dễ quản lý và tái sử dụng.
-	**JSX Syntax**: Kết hợp JavaScript và HTML trong cùng một file, giúp code dễ đọc và bảo trì.
-	**React Hooks**: useState, useEffect, useContext... giúp quản lý state và side effect trong functional component.
-	**React Router**: Thư viện routing cho Single Page Application (SPA).
-	**Ecosystem phong phú**: Nhiều thư viện hỗ trợ như Axios (HTTP client), React Query (data fetching), Zustand/Redux (state management).

2.2.2.2	Kiến trúc hệ thống

Hệ thống DAVictory sử dụng kiến trúc **Client-Server** với sự phân tách rõ ràng giữa Frontend và Backend:

**Backend (Spring Boot)**
-	**Controller Layer**: Xử lý HTTP request từ client, gọi Service layer và trả về response.
-	**Service Layer**: Chứa logic nghiệp vụ chính (tạo đề thi, chấm điểm, quản lý lớp học...).
-	**Repository Layer**: Tương tác với cơ sở dữ liệu thông qua Spring Data JPA.
-	**Entity Layer**: Định nghĩa các entity tương ứng với bảng trong database.
-	**DTO Layer**: Data Transfer Object để truyền dữ liệu giữa các layer.
-	**Security Layer**: Xác thực JWT, phân quyền RBAC (Role-Based Access Control).

**Frontend (React)**
-	**Components**: Các component UI như TestCard, QuestionItem, ExamAttemptPage...
-	**Pages**: Các trang chính như HomePage, TestBuilderPage, ExamPage, DashboardPage...
-	**Services/API**: Axios client để gọi API từ Backend.
-	**Hooks**: Custom hooks để quản lý state và logic tái sử dụng.
-	**Context/State Management**: Quản lý state toàn cục (user info, auth token...).
-	**Router**: React Router để điều hướng giữa các trang.

**Luồng dữ liệu:**
1.	User tương tác với giao diện React (Frontend).
2.	Frontend gửi HTTP request (GET/POST/PUT/DELETE) đến Backend API.
3.	Backend Controller nhận request, validate dữ liệu.
4.	Controller gọi Service để xử lý logic nghiệp vụ.
5.	Service gọi Repository để truy xuất/cập nhật database.
6.	Repository trả kết quả về Service, Service xử lý và trả về Controller.
7.	Controller trả response (JSON) về Frontend.
8.	Frontend nhận response, cập nhật UI.

2.2.2.3	Các công nghệ và thư viện chính

**Backend:**
-	**Spring Boot 4.0.3**: Framework chính.
-	**Spring Data JPA**: ORM để tương tác với MySQL.
-	**Spring Security**: Bảo mật và phân quyền.
-	**JWT (JSON Web Token)**: Xác thực người dùng.
-	**MySQL Connector**: Driver kết nối MySQL.

-	**Google Drive API**: Lưu trữ file media (audio, image).

```java
@Service
public class ExamAttemptService {
    @Transactional
    public ExamAttempt startAttempt(ExamAttemptStartRequest request) {
        ExamAttempt attempt = new ExamAttempt();
        attempt.setUserId(request.getUserId());
        attempt.setTestId(request.getTestId());
        attempt.setStatus(AttemptStatus.IN_PROGRESS);
        attempt.setStartedAt(LocalDateTime.now());
        
        // Tạo AttemptAnswer cho mỗi câu hỏi
        List<Question> questions = questionRepository.findByTestId(request.getTestId());
        for (Question question : questions) {
            AttemptAnswer answer = new AttemptAnswer();
            answer.setExamAttemptId(attempt.getId());
            answer.setQuestionId(question.getId());
            answer.setIsAnswered(false);
            attemptAnswerRepository.save(answer);
        }
        
        return examAttemptRepository.save(attempt);
    }
    
    @Transactional
    public ExamAttempt submitAttempt(ExamAttemptSubmitRequest request) {
        ExamAttempt attempt = examAttemptRepository.findById(request.getAttemptId());
        attempt.setStatus(AttemptStatus.SUBMITTED);
        attempt.setSubmittedAt(LocalDateTime.now());
        
        // Chấm điểm tự động cho Listening và Reading
        gradeAttempt(attempt);
        
        return examAttemptRepository.save(attempt);
    }
    
    private void gradeAttempt(ExamAttempt attempt) {
        List<AttemptAnswer> answers = attemptAnswerRepository.findByExamAttemptId(attempt.getId());
        int correctCount = 0;
        
        for (AttemptAnswer answer : answers) {
            boolean isCorrect = isAnswerCorrect(answer);
            answer.setIsCorrect(isCorrect);
            if (isCorrect) correctCount++;
        }
        
        // Tính band score theo bảng quy đổi IELTS
        double bandScore = calculateBandScore(correctCount, answers.size());
        attempt.setBandScore(bandScore);
    }
}
```

```java
    public void gradeWriting(WritingGradeRequest request) {
        StudentWritingSubmission submission = writingRepository.findById(request.getSubmissionId());
        
        // Tính band score trung bình từ 4 tiêu chí
        double avgScore = (request.getTaskAchievement() +

                          request.getGrammar()) / 4.0;
        
        submission.setBandScore(roundBandScore(avgScore));
```
