CHƯƠNG 1: TỔNG QUAN ĐỀ TÀI
================================================================================

1.1. Lý do chọn đề tài
    Nội dung:
    - Nhu cầu luyện thi IELTS ngày càng tăng tại Việt Nam và tại Vĩnh Long
    - Trung tâm Ngoại ngữ - Tin học Victory gặp khó khăn trong việc:
        + Quản lý đề thi thủ công (file Word/PDF rời rạc)
        + Tổ chức thi trực tiếp tốn thời gian, chi phí
        + Chấm điểm Writing/Speaking mất nhiều thời gian
        + Theo dõi tiến độ học viên không tập trung
        + Thiếu công cụ để học viên tự luyện tập tại nhà
    - Các hệ thống hiện có (IELTS Online, British Council) có chi phí cao
    - Xu hướng ứng dụng AI trong giáo dục, đặc biệt là chấm điểm tự động

    Liên hệ source:
    - Dựa trên khảo sát thực tế (ghi nhận trong yêu cầu phần mềm từ trung tâm)
    - Các module Class, Exam, Assignment trong code phản ánh nhu cầu quản lý

1.2. Mục tiêu đề tài
    Mục tiêu tổng quát:
    - Xây dựng hệ thống website hỗ trợ luyện thi IELTS trực tuyến cho 
      Trung tâm Ngoại ngữ - Tin học Victory

    Mục tiêu cụ thể:
    - Xây dựng hệ thống quản lý người dùng với 5 vai trò: 
      Guest, Student, Teacher, Manager, Admin
    - Xây dựng module quản lý lớp học, phân công giáo viên
    - Xây dựng module tạo đề thi với đầy đủ 4 kỹ năng 
      Listening, Reading, Writing, Speaking
    - Xây dựng module tổ chức thi và làm bài trực tuyến
    - Xây dựng module chấm điểm Writing bằng AI
    - Xây dựng module chấm điểm Speaking bằng AI (STT + NLP)
    - Xây dựng module quản lý bài tập, giao bài cho học viên
    - Xây dựng module import đề thi tự động (PDF/image → test structure)
    - Xây dựng AI Agent hỗ trợ tạo nội dung, báo cáo
    - Xây dựng module thống kê và báo cáo

    Liên hệ source:
    - Backend: 32 controllers, 26+ services, 71 entities
    - Frontend: 49 pages, 29 common components
    - 4 AI services: ai-writing-python, ai-speaking-python, ai-import-python, ai-agent-python

1.3. Đối tượng và phạm vi nghiên cứu
    Đối tượng:
    - Học viên: đăng ký, làm bài thi, nộp bài, xem kết quả, theo dõi tiến độ
    - Giáo viên: tạo đề thi, chấm điểm, quản lý lớp, giao bài tập
    - Quản trị viên: quản lý người dùng, lớp học, hệ thống
    - Khách: làm bài thi thử qua link chia sẻ

    Phạm vi:
    - Phạm vi chức năng: 4 kỹ năng IELTS (Listening, Reading, Writing, Speaking)
    - Phạm vi công nghệ: Spring Boot, React, MySQL, microservices AI
    - Phạm vi địa lý: Trung tâm Victory tại Vĩnh Long
    - Giới hạn: chưa triển khai payment, chưa có mobile app

    Liên hệ source:
    - GuestExamAttempt entity cho khách làm bài
    - Role enum: GUEST, STUDENT, TEACHER, MANAGER, ADMIN
    - SecurityConfig phân quyền từng endpoint

1.4. Phương pháp thực hiện
    - Phương pháp khảo sát: phỏng vấn trực tiếp, thu thập yêu cầu
    - Phương pháp phân tích: sử dụng biểu đồ UML (Use Case, Sequence, Activity)
    - Phương pháp thiết kế: CSDL quan hệ, REST API, kiến trúc microservices
    - Phương pháp lập trình: 
        + Backend: Spring Boot (Java 21), Spring Data JPA, Spring Security
        + Frontend: React 19, Vite 8, Axios
        + AI Services: Python FastAPI, Groq API, OpenAI API
    - Phương pháp kiểm thử: kiểm thử thủ công API, kiểm thử giao diện
    - Phương pháp đánh giá: so sánh kết quả chấm AI với chấm tay

1.5. Ý nghĩa thực tiễn của đề tài

    - AI hỗ trợ chấm điểm Writing/Speaking

    Đối với học viên:
    - Luyện thi mọi lúc mọi nơi
    - Nhận phản hồi chi tiết từ AI
    - Theo dõi lịch sử và tiến bộ

CHƯƠNG 2: CƠ SỞ LÝ THUYẾT VÀ CÔNG NGHỆ SỬ DỤNG
================================================================================

2.1. Tổng quan về bài thi IELTS
    2.1.1. Cấu trúc bài thi IELTS
        - 4 kỹ năng: Listening (30 phút), Reading (60 phút), 
          Writing (60 phút), Speaking (11-14 phút)
        - Thang điểm 9.0 (từng kỹ năng và overall)
    2.1.2. Phân loại: Academic và General Training
    2.1.3. Các dạng câu hỏi thường gặp
        - Multiple Choice, True/False/Not Given, Matching Headings
        - Sentence Completion, Summary Completion, Short Answer
        - Flow Chart, Diagram Labeling
        - Writing Task 1 (biểu đồ/thư), Task 2 (essay)
    
    Liên hệ source:
    - Session entity: 4 skill types (LISTENING, READING, WRITING, SPEAKING)
    - QuestionType enum: 14+ loại câu hỏi
    - DifficultyLevel: 5 cấp độ từ Beginner đến Advanced

2.2. Hệ thống đào tạo trực tuyến (E-learning)
    2.2.1. Khái niệm và lợi ích
    2.2.2. Mô hình phổ biến: LMS, MOOC
    2.2.3. Các hệ thống tương tự
        - IELTS Online của British Council
        - IELTS Liz, IELTS Advantage
        - Các nền tảng khác (khoá học tính phí)

    Liên hệ source:
    - Module LMS trong frontend (LmsTeacherDashboard, StudentLms)
    - Assignment, Class, Center entities

2.3. AI trong giáo dục
    2.3.1. Natural Language Processing (NLP)
        - Xử lý ngôn ngữ tự nhiên
        - Ứng dụng trong chấm bài Writing
    2.3.2. Speech Recognition
        - Chuyển giọng nói thành văn bản (STT)
        - Ứng dụng trong chấm bài Speaking
    2.3.3. Large Language Models (LLM)
        - GPT-4o, Groq Qwen, Llama
        - Ứng dụng trong sinh câu hỏi, chấm điểm, tạo nội dung
    2.3.4. Retrieval-Augmented Generation (RAG)
        - Kết hợp truy xuất thông tin + sinh văn bản
        - ChromaDB vector search

    Liên hệ source:
    - ai-writing-python: ChromaDB RAG cho grading
    - ai-speaking-python: Whisper STT + LLM scoring
    - ai-agent-python: LLM agents (content, info, report, email)

2.4. Công nghệ Backend
    2.4.1. Ngôn ngữ Java
        - Lịch sử phát triển, đặc điểm
        - Java 21: virtual threads, pattern matching, records
    2.4.2. Spring Boot 4.0.3
        - Spring Framework, Inversion of Control (IoC)
        - Spring MVC: mô hình Model-View-Controller
        - Spring Data JPA: ORM, repository pattern
        - Spring Security: xác thực, phân quyền
        - Spring Cache: caching dữ liệu
        - Ưu điểm: cấu hình tự động, giảm boilerplate
    2.4.3. REST API
        - Nguyên tắc RESTful
        - HTTP methods: GET, POST, PUT, DELETE
        - Định dạng JSON, mã trạng thái HTTP
    2.4.4. JWT (JSON Web Token)
        - Cấu trúc token: header, payload, signature
        - HMAC-SHA, access token (24h), refresh token (7 ngày)
        - Xác thực không trạng thái (stateless)

    Liên hệ source:
    - pom.xml: Spring Boot 4.0.3, Java 21, jjwt 0.12.6
    - SecurityConfig: filter chain, JwtAuthenticationFilter
    - JwtUtil: generate/validate token
    - application.yaml: jwt.secret, jwt.expiration

2.5. Công nghệ Frontend
    2.5.1. React 19
        - Component-based architecture
        - JSX, Virtual DOM, Hooks
        - Lazy loading, Suspense
    2.5.2. Vite 8
        - Build tool nhanh, HMR
        - Proxy configuration cho API
    2.5.3. React Router v7
        - Routing, nested routes
        - Lazy loading routes
    2.5.4. Axios
        - HTTP client, interceptors
        - Xử lý token, error handling
    2.5.5. Thư viện hỗ trợ
        - Framer Motion (animation)
        - Lucide React (icons)
        - TipTap, React Quill (rich text)
        - react-markdown
        - @dnd-kit (drag & drop)

    Liên hệ source:
    - package.json: react 19.2.0, vite 8.x
    - vite.config.js: proxy /api -> localhost:8080
    - services/authApi.js: axios interceptors

2.6. Cơ sở dữ liệu MySQL
    2.6.1. Hệ quản trị CSDL quan hệ MySQL
    2.6.2. JPA/Hibernate ORM
        - Ánh xạ đối tượng-quan hệ
        - JPA repositories
        - Cascade types, Fetch types
    2.6.3. Connection Pooling với HikariCP

    Liên hệ source:
    - application.yaml: spring.datasource, HikariCP max 20
    - 71 JPA entities, 71 repositories
    - ddl-auto: update

2.7. Python và FastAPI (AI Microservices)
    2.7.1. Fast Framework
        - ASGI, async/await
        - Pydantic models
    2.7.2. Các thư viện AI
        - httpx, tenacity, loguru
        - chromadb (vector database)
        - pymupdf, pdfplumber, pytesseract (document processing)
    2.7.3. Docker
        - Dockerfile, docker-compose
        - Triển khai microservices

    Liên hệ source:
    - 4 thư mục: ai-agent-python, ai-import-python, 
      ai-speaking-python, ai-writing-python
    - docker-compose.yml: redis + ai-speaking

2.8. Google Drive API
    2.8.1. OAuth 2.0 authentication
    2.8.2. Upload/download files
    2.8.3. Quản lý thư mục

    Liên hệ source:
    - application.yaml: google.drive.*
    - GoogleDriveOAuth2Service
    - FileUploadService

        - Quy mô: nhiều lớp học, nhiều giáo viên
    3.1.2. Quy trình hiện tại
        - Giáo viên soạn đề thi trên Word/PDF
        - In ấn đề thi, tổ chức thi trực tiếp
        - Chấm điểm thủ công
        - Quản lý điểm bằng Excel
    3.1.3. Khó khăn và hạn chế
        - Tốn thời gian, công sức
        - Khó quản lý tập trung
        - Học viên khó tự luyện tại nhà
        - Không có số liệu thống kê

3.2. Yêu cầu chức năng
    (Liệt kê chi tiết từng chức năng theo module, có mã số)

    3.2.1. Module xác thực và phân quyền (AUTH)
        AUTH-01: Đăng ký tài khoản (POST /api/auth/register)
        AUTH-02: Đăng nhập, nhận JWT (POST /api/auth/login)
        AUTH-03: Đăng xuất (POST /api/auth/logout)
        AUTH-04: Xem thông tin cá nhân (GET /api/auth/me)
        AUTH-05: Phân quyền theo 5 vai trò (GUEST/STUDENT/TEACHER/MANAGER/ADMIN)
    
    3.2.2. Module quản lý người dùng (USER)
        USER-01: Xem danh sách người dùng (GET /api/users)
        USER-02: Xem chi tiết người dùng (GET /api/users/{id})
        USER-03: Cập nhật thông tin cá nhân (PUT /api/users/me)
        USER-04: Đổi mật khẩu (PUT /api/users/me/password)
        USER-05: Thêm/xóa vai trò (POST/DELETE /api/users/{id}/roles/{roleName})
        USER-06: Xóa người dùng (DELETE /api/users/{id})
        USER-07: Import học viên từ CSV (POST /api/admin/users/import-students)
        USER-08: Quản trị: dashboard stats, toggle active, đổi mật khẩu

    3.2.3. Module quản lý lớp học (CLASS)
        CLASS-01: Xem danh sách lớp học
        CLASS-02: Tạo lớp, cập nhật, xóa lớp
        CLASS-03: Thêm/xóa học viên vào lớp
        CLASS-04: Phân công giáo viên cho lớp
        CLASS-05: Học viên xem lớp của mình
        CLASS-06: Giáo viên xem lớp được phân công

    3.2.4. Module quản lý đề thi (TEST)
        TEST-01: Tạo đề thi mới (POST /api/test-builder/save-full)
        TEST-02: Chỉnh sửa đề thi (GET/PUT test-builder)
        TEST-03: Xóa đề thi (soft delete / permanent)
        TEST-04: Thay đổi trạng thái đề thi (DRAFT→REVIEWING→PUBLISHED→ARCHIVED)
        TEST-05: Thêm Session (Listening/Reading/Writing/Speaking)
        TEST-06: Thêm Part (Part 1, Part 2,...) vào Session
        TEST-07: Thêm Question Group (kèm passage/audio)
        TEST-08: Thêm câu hỏi các loại (MCQ, TFNG, Matching,...)
        TEST-09: Xem lịch sử phiên bản đề thi
        TEST-10: Tạo đề thi ngẫu nhiên từ ngân hàng câu hỏi
        TEST-11: Tạo share link cho khách

    3.2.5. Module tổ chức thi và làm bài (EXAM)
        EXAM-01: Tạo kỳ thi (POST /api/exams)
        EXAM-02: Bắt đầu/mở/đóng kỳ thi
        EXAM-03: Xác thực mật khẩu kỳ thi
        EXAM-04: Bắt đầu làm bài (POST /api/exam-attempts/start)
        EXAM-05: Lưu bài tạm thời (auto-save 30 giây)
        EXAM-06: Nộp bài (POST /api/exam-attempts/{id}/submit)
        EXAM-07: Auto-submit khi hết giờ
        EXAM-08: Lưu backup câu trả lời trong quá trình làm
        EXAM-09: Chấm điểm tự động (listening/reading)
        EXAM-10: Lịch sử các lần làm bài
        EXAM-11: Khách làm bài qua share link

    3.2.6. Module Writing (WRITING)
        WRITING-01: Nộp bài Writing (POST /api/writing/submit)
        WRITING-02: Xem bài Writing đã nộp
        WRITING-03: Giáo viên chấm điểm Writing (POST /api/writing/grade)
        WRITING-04: AI chấm điểm Writing (POST /api/writing/ai-grade)
        WRITING-05: Xem kết quả chấm AI chi tiết (4 tiêu chí: TA, CC, LR, GRA)
        WRITING-06: Phê duyệt/từ chối điểm AI

    3.2.7. Module Speaking (SPEAKING)
        SPEAKING-01: Tạo phiên Speaking
        SPEAKING-02: Ghi âm và nộp bài Speaking
        SPEAKING-03: Giáo viên chấm điểm Speaking
        SPEAKING-04: AI chấm điểm Speaking (4 tiêu chí: FC, LR, GRA, P)
        SPEAKING-05: Tạo đề Speaking động từ frames/combos

    3.2.8. Module bài tập (ASSIGNMENT)
        ASM-01: Tạo bài tập từ đề thi có sẵn
        ASM-02: Giao bài tập cho lớp
        ASM-03: Học viên nộp bài tập
        ASM-04: Giáo viên chấm bài tập
        ASM-05: Xem danh sách bài nộp

    3.2.9. Module AI Import (AI_IMPORT)
        AI_IMPORT-01: Upload file PDF/DOCX/TXT/image
        AI_IMPORT-02: Parse và trích xuất nội dung
        AI_IMPORT-03: AI phân tích và tạo cấu trúc đề thi
        AI_IMPORT-04: Lưu đề thi vào hệ thống

    3.2.10. Module AI Agent (AI_AGENT)
        AI_AGENT-01: Chat với AI Agent (hỏi đáp thông tin)
        AI_AGENT-02: Tạo bài viết blog (wizard: topic→outline→content→publish)
        AI_AGENT-03: Tạo báo cáo thống kê
        AI_AGENT-04: Quản lý danh mục bài viết
        AI_AGENT-05: Phê duyệt hành động (human-in-the-loop)

    3.2.11. Module quản lý file (FILE)
        FILE-01: Upload file lên Google Drive
        FILE-02: Xem trước file
        FILE-03: Xóa file

    3.2.12. Module thống kê (STATS)
        STATS-01: Dashboard quản trị
        STATS-02: Theo dõi tiến độ học viên (StudentProgress)
        STATS-03: Thống kê điểm theo kỹ năng (StudentSkillScore)
        STATS-04: Lịch sử làm bài

3.3. Yêu cầu phi chức năng
    - Bảo mật: xác thực JWT, phân quyền endpoint, BCrypt mã hóa mật khẩu
    - Hiệu năng: caching (Spring Cache), HikariCP connection pool
    - Khả năng mở rộng: microservices AI độc lập
    - Khả dụng: xử lý timeout, auto-save, backup
    - Giao diện: responsive, hỗ trợ mọi thiết bị
    - Bảo mật dữ liệu: soft delete (deletedAt), không xóa cứng dữ liệu

    Liên hệ source:
    - SecurityConfig: endpoint matchers
    - @EnableCaching
    - Soft delete trong Test (DELETED status), User (deletedAt)

3.4. Tác nhân hệ thống (Actors)
    - Guest (khách): làm bài qua share link, không cần đăng nhập
    - Student (học viên): đăng ký, làm bài, xem kết quả, nộp bài tập
    - Teacher (giáo viên): tạo đề, chấm điểm, quản lý lớp, giao bài
    - Manager (quản lý): quản lý lớp, xem báo cáo, AI Agent
    - Admin (quản trị): quản lý người dùng, vai trò, hệ thống

3.5. Biểu đồ Use Case (tổng thể)
